# 工程质量说明（Phase 1）

> 项目：考勤填报系统（Web）  
> 代码路径：`/Users/liyifan/Desktop/attendance-report`

## 1. 目标与范围

本项目聚焦 Phase 1 最小可用闭环：

- 员工：登录、日次填报、历史查看、月度提交
- 管理：员工查看、提交情况、按日编辑、导出 CSV/Excel/PDF
- 核心逻辑：总工时计算、平日/休日/法定节假日标记、提交状态管理

未纳入 Phase 1：语音/拍照导入、审批流、工资系统对接。

---

## 2. 技术架构

### 2.1 技术栈

- 前端/服务端：Next.js 15（App Router）+ React 19 + TypeScript
- 数据访问：Prisma ORM
- 数据库：SQLite（开发环境）
- 会话认证：JWT（`jose`）+ HttpOnly Cookie
- 密码哈希：`bcryptjs`
- 导出：`xlsx`（Excel）、`pdf-lib` + `@pdf-lib/fontkit`（PDF）

### 2.2 分层说明

- `src/app/*`：页面与 Route Handler（含`/api/export`）
- `src/lib/*`：数据库、认证、工时计算、导出映射与PDF绘制
- `prisma/schema.prisma`：数据模型定义
- `scripts/ensure-pdf-font.mjs`：PDF字体预下载脚本

---

## 3. 数据模型质量

来自 `prisma/schema.prisma` 的核心对象：

- `User`：账号与角色（`EMPLOYEE` / `ADMIN`）
- `Employee`：员工主数据（工号、姓名、部门）
- `AttendanceRecord`：日考勤明细（唯一键：`employeeId + workDate`）
- `MonthlySubmission`：月提交状态（唯一键：`employeeId + yearMonth`）
- `Holiday`：节假日主数据

关键枚举：

- `AttendanceStatus`：`PRESENT` / `LEAVE`
- `WorkDayType`：`WEEKDAY` / `WEEKEND_WORK` / `PUBLIC_HOLIDAY_WORK`
- `SubmissionStatus`：`DRAFT` / `SUBMITTED`

数据约束特点：

- 同一员工同一天仅一条记录，避免重复填报
- 月提交与日记录分离，便于月末流程管理
- 通过 ORM 统一访问，降低 SQL 注入风险

## 4. 业务规则实现

### 4.1 工时计算

`computeTotalWorkMinutes()` 规则：

- 休假/缺勤：工时=0
- 正常出勤：`(下班-上班)-扣除分钟`
- 下班早于上班：按跨日处理（+24h）
- 工时最小为0（防止负值）

### 4.2 提交流程

- 员工提交月度后，`MonthlySubmission.status = SUBMITTED`
- 同月 `AttendanceRecord.submissionStatus` 批量更新为 `SUBMITTED`
- 员工端禁止再修改当月记录；管理员仍可修正

### 4.3 导出规则

- CSV/Excel/PDF 全部使用中文表头与中文枚举值
- 员工导出仅本人范围；管理员可导出全员或单员工
- PDF为A4横向表格，中文字体完整嵌入以保证可读性

## 5. 安全与权限

### 5.1 认证

- 登录后签发 JWT（有效期7天）
- 会话通过 HttpOnly Cookie 存储，减少前端窃取风险

### 5.2 鉴权

- 中间件拦截未登录访问，并按角色重定向
- 员工无法访问 `/admin/*`
- 管理员无法访问 `/employee/*`
- 导出接口二次校验角色与范围（员工不能导出他人数据）

### 5.3 输入校验

- 日期、年月格式正则校验
- 出勤状态为“正常出勤”时必须填写上下班时间
- 数值字段（扣除分钟）做非负处理

## 6. 可运维性

### 6.1 启动与构建

```bash
npm install
npx prisma db push
npx prisma db seed
npm run dev
npm run build
```

### 6.2 PDF 字体保障

- `postinstall` 自动执行 `scripts/ensure-pdf-font.mjs`
- 支持 `npm run pdf-font` 手动拉取字体
- 支持 `CJK_FONT_PATH` 指定本地字体文件

### 6.3 已知限制

- 当前数据库为 SQLite，不适合高并发生产写入
- 未接入集中日志、APM、告警
- 审计日志尚未独立建模（建议下阶段补充）
